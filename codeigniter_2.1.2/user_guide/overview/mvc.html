Model-View-Controller

O CodeIgniter é baseado no padrão Model-View-Controller. O MVC é uma abordagem de desenvolvimento que separa a lógica do aplicativo da apresentação. Na prática, isso permite que suas páginas web contenham o mínimo de scripts já que a apresentação é separada dos scripts PHP.

  • O Model representa suas estruturas de dados. Normalmente, suas classes de Model irão conter funções que ajudam a recuperar, inserir e atualizar as informações em seu banco de dados.
  • A View é a informação que é apresentada ao usuário. A View será normalmente uma página web, mas no CodeIgniter uma view pode também ser um fragmento de página como um rodapé ou cabeçalho. Também pode ser uma página RSS ou diversos outros tipos de "páginas".
  • O Controller funciona como um intermediário entre o Model, a View e qualquer outro recurso necessário para processar a requisição HTTP e gerar uma página web.

CodeIgniter has a fairly loose approach to MVC since Models are not required. If you don't need the added separation, or find that maintaining models requires more complexity than you want, you can ignore them and build your application minimally using Controllers and Views. CodeIgniter also enables you to incorporate your own existing scripts, or even develop core libraries for the system, enabling you to work in a way that makes the most sense to you.

O CodeIgniter tem uma abordagem de MVC bastante flexível pois os Models não são necessários. Se você não precisar da separação adicional, ou achar que a manutenção de Models requerem maior complexidade do que você quer, você pode ignorá-los e construir sua aplicação usando apenas Controllers e Views. O CodeIgniter também permite que você incorpore seus próprios scripts, ou mesmo desenvolva bibliotecas para o núcleo do sistema, permitindo que você trabalhe de uma forma que faça mais sentido para você.
