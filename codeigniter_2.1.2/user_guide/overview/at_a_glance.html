Uma olhada rápida no CodeIgniter

O CodeIgniter é um Framework de Aplicativo

O CodeIgniter é um kit de ferramentas para quem desenvolve aplicações web usando PHP. Seu objetivo é permitir que você desenvolva projetos mais rapidamente do que você desenvolveria se estivesse escrevendo código a partir do zero, proporcionando um rico conjunto de bibliotecas necessárias para as tarefas mais comuns, bem como uma interface simples e estrutura lógica para acessar essas bibliotecas. O CodeIgniter permite que você mantenha o foco em seu projeto, minimizando a quantidade de código necessário para uma dada tarefa.

O CodeIgniter é Livre

O CodeIgniter é licenciado sob uma licença Open Source no estilo Apache/BSD-style para que você possa usá-lo como quiser. Para mais informações, por favor leira o contrato de licença.

O CodeIgniter é Leve

Realmente leve. O núcleo do sistema requer apenas algumas bibliotecas muito pequenas. Isto está em contraste com alguns frameworks que requerem significativamente mais recursos. Bibliotecas adicionais são carregadas dinamicamente sob requisição, de acordo com suas necessidades para um determinado processo, de modo que o sistema principal é muito magro e rápido.

O CodeIgniter é Rápido

Realmente rápido. Nós o desafiamos a encontrar um framework que tenha um desempenho melhor que o CodeIgniter.

O CodeIgniter usa M-V-C

CodeIgniter usa a abordagem Modelo-Visão-Controle (Model-View-Controller), que permite a ótima separação entre a lógica e apresentação. Isto é particularmente bom para projetos em que designers estão trabalhando com seus arquivos de template, assim, o código nestes arquivos será minimizado. Nós explicamos a abordagem MVC em mais detalhes em sua página específica.

O CodeIgniter Gera URLs Amigáveis

As URLs geradas pelo CodeIgniter são limpas e amigáveis à motores de busca. Ao invés de usar a abordagem de "query string" padrão para as URLs que é sinônimo de sistemas dinâmicos, o CodeIgniter usa uma abordagem baseada em segmentos:

example.com/news/article/345

Nota: Por padrão o arquivo index.php é incluído na URL mas ele pode ser removido usando um simples arquivo .htacces.

O CodeIgniter Empacota Grandes Recursos

O CodeIgniter vem com uma grande variedade de bibliotecas que permitem as tarefas de desenvolvimento mais utilizados na web, como acessar um banco de dados, envio de e-mail, validar os dados do formulário, manter sessões, manipular imagens, trabalhar com dados XML-RPC e muito mais.

O CodeIgniter é Extensível

O sistema pode ser facilmente extendido através do uso de bibliotecas, helpers, ou através de classes extendidas ou mesmo de hooks do sistema.

O CodeIgniter Não Exige um Motor de Template

Embora o CodeIgniter venha com um simples parser de template que pode ser usado opicionalmente, ele não tem seu uso obrigatório. Os motores de template simplesmente não têm a mesma performance do PHP nativo, e a sintaxe que deve ser aprendida para usar um motor de template é geralmente só um pouco mais fácil que aprender do que o básico de PHP. Considere este bloco de código PHP: <ul>

<?php foreach ($addressbook as $name):?>

<li><?=$name?></li>

<?php endforeach; ?>

</ul>

Comparando-o a esse pseudo-código usado por um motor de template:

<ul>

{foreach from=$addressbook item="name"}

<li>{$name}</li>

{/foreach}

</ul>

Sim, o segundo exemplo é um pouco mais claro, mas ele tem o preço do desempenho, pois o pseudo-código deve ser convertido em PHP para executar. Como um de nossos objetivos é desempenho máximo , optamos por não requerer o uso de um motor de template.

Programadores adoram codificar e odeiam documentar. Nós não somos diferentes, é claro, mas como a documentação é tão importante como o próprio código, estamos empenhados em fazê-lo. Nosso código-fonte é extremamente limpo e bem comentado também.

O CodeIgniter tem uma Comunidade de Usuários Colaboradora

Nossa crescente comunidade de usuários pode ser vistaparticipando ativamente em nosso Fórum da Comunidade.
