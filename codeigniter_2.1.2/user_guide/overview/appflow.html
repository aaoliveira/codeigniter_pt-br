Fluxograma da Aplicação

O gráfico a seguir ilustra como os dados fluem por todo o sistema:

  1. O arquivo index.php serve como controlador principal, inicializando os recursos básicos necessários para executar o CodeIgniter.
  2. O Router examina a requisição HTTP para determinar o que deve ser feito com ele.
  3. Se um arquivo de cache existe, ele é enviado diretamente para o navegador, ignorando a execução normal do sistema.
  4. Segurança. Antes do Controller do aplicativo ser carregado, a requisição HTTP e quaisquer dados enviados pelo usuário são filtrados por segurança.
  5. O Controller carrega o Model, as bibliotecas principais, auxiliares e quaisquer outros recursos necessários para processar a solicitação específica.
  6. Por fim, a View é renderizada em seguida sendo enviada para o navegador web para ser vista. Se o cache está habilitado, a View é armazenada em cache primeiro para que nas solicitações subseqüentes, possa ser entregue.
